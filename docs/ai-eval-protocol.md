# Bluebird AI Eval Protocol

## 목적
Bluebird Master Protocol이 실제 분석 응답에 반영되는지 지속적으로 검증한다.

## 평가 범위
- 왜곡 탐지 정확도(5대 왜곡 taxonomy 준수)
- 프레임 추론(`frame_type`) 일관성
- 준거점/확률/CAS 신호 추출 여부
- 소크라테스 질문의 시스템2 기동 품질
- JSON 구조 무결성

## 테스트 데이터
- 파일: `lib/ai/eval-cases.ts`
- 기본 10개 케이스 사용
- 릴리즈 전 최소 1회 전체 실행

## MVP 통과 기준 (Pass/Fail)
1. 10개 케이스 중 8개 이상에서 기대 왜곡 또는 수용 가능한 근접 왜곡 탐지
2. 질문 3개 모두 숫자/확률/비율 입력을 유도
3. JSON 파싱 실패율 0%
4. API 실패 시 폴백으로 UI 중단 없음

## 운영 루프 (Build-Measure-Learn)
1. **Build**
   - 프롬프트/스키마/가드레일 변경 배포
2. **Measure**
   - eval 케이스 결과를 체크리스트로 채점
   - 실패 사례를 유형별로 분류
3. **Learn**
   - 반복 실패 패턴에 맞춰 few-shot/정규화 규칙 수정
   - 수정 후 동일 케이스 재검증

## 체크리스트
- [ ] 왜곡 라벨이 기대값과 일치하는가
- [ ] `frame_type`이 손실/이득/혼합 분류 논리와 맞는가
- [ ] `reference_point`가 문맥에서 추론 가능하게 작성됐는가
- [ ] `probability_estimate`가 0~100 범위인가
- [ ] `cas_signal` 값이 0~1 범위인가
- [ ] 질문이 감정 위로 대신 계량 검증을 유도하는가

## 실패 대응 원칙
- 모델 과부하(429/503): 재시도 후 폴백 응답으로 진행 유지
- JSON 파싱 실패: 정규화 파서 + 기본 스키마 채움
- 필드 누락: zod 검증으로 기본값 보정
