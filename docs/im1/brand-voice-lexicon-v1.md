# BlueBird Brand Voice Lexicon — v1 (2026-05-15)

> **소유:** content-marketer (CMO 산하). **출처:** 5/15 weekly all-hands §5 #F-2.
> **목적:** BlueBird의 분석가 톤을 단일 어휘 사전으로 명문화. 모든 사용자 노출 카피(앱·이메일·SMS·소셜)는 본 문서를 단일 출처로 한다. 코드 가드와 일대일 매핑 (`scripts/lint-copy.ts`).

---

## 0. 정체성 — 한 줄 정의

> **"BlueBird는 위로 챗봇이 아니라 분석 도구다. 카피는 분석가가 데이터를 보고 말하듯 쓴다."**

운영 원칙 4종 (`lib/ai/bluebird-protocol.ts BLUEBIRD_OPERATING_PRINCIPLES`):

1. **데이터 우선** — 주관적 고통을 수치·비교 가능한 데이터로 객관화
2. **감정 배제** — 위로 대신 논리 무결성·사고 검증
3. **자율성 지향** — 정답 제시 ❌, 시스템2 기동 질문 ✓
4. **경직성 타파** — 사실처럼 보이는 생각도 행동 제약 시 인지적 틈새 탐색

→ 카피 작성 시 위 4 원칙 중 최소 1개를 강화해야 한다. 4 원칙과 무관·반대인 어휘는 모두 금지.

## 1. 카테고리별 금지 어휘 (lint-copy 자동 차단)

본 섹션의 모든 패턴은 `scripts/lint-copy.ts`에 코드로 강제되어 있다. PR 머지 전 `npm run lint:copy`가 자동 검증.

### 1.1 위로·응원 어휘 (COMFORT_PATTERNS) — 11 패턴

| 금지 어휘 | 왜 차단 | 대체 |
|---|---|---|
| `괜찮아요`, `괜찮아` | 가치 판단 — 데이터 우선 위반 | "관찰됐어요", "기록됐어요" |
| `힘내세요` | 감정 응원 — 운영 원칙 #2 위반 | (불필요) |
| `잘하고 있어` | 가치 판단 | "체크인 5일 연속" 같은 사실 진술 |
| `응원해요/합니다/드려요` | 감정 응원 | (불필요) |
| `안녕!` | 캐릭터 친근체 | "안녕하세요" (중립체) |
| `함께 해/들여다/가봐/시작` | 동반체 — 자율성 위반 | "관찰하세요", "확인하세요" |
| `친구처럼` | 친구체 | "분석 도구처럼" |
| `마음을 안아` | 위로 메타포 | "사고를 관찰" |
| `오늘 하루도` | 응원체 | (불필요) |
| `소중한/해` | 정서 어휘 | "측정 가능한", "관찰 가능한" |
| `❤️`, `🌟` | 정서 이모지 | (이모지 사용 금지) |

### 1.2 항해 메타포 (NAUTICAL_PATTERNS) — 9 패턴

> **상태:** 인터뷰 검증(B3) 대기 — 부분 잔존 허용. 신규 카피에는 사용 금지.

| 금지 어휘 | 대체 |
|---|---|
| `항해` | "분석", "관찰", "기록" |
| `나침반` | "지표", "기준점" |
| `안개` (`짙은 안개`, `안개 지역`, `안개 등급` 제외) | "불확실성" |
| `별빛` | "신호" |
| `바다` (단독 사용) | (불필요) |
| `✦`, `✧`, `★`, `☆` 글리프 | (사용 금지) |

기존 사용처 4건 (`app/analyze/[id]/page.tsx`·`app/auth/signup/page.tsx`·`app/checkin/page.tsx`) — B3 인터뷰 후 일괄 정리 예정.

### 1.3 의료기기·임상 표현 (MEDICAL_PATTERNS) — 6 패턴

> **법적 가드레일.** 신규 카피에 등장 시 즉시 차단 (whitelist 영역 제외: `app/disclaimer`·`app/terms`·`app/safety/resources` — 법적 면책 의도적 사용).

| 금지 어휘 | 대체 |
|---|---|
| `교정` | "조정", "변화" |
| `진단` | "분석", "분류" |
| `치료` (인지행동`치료` `치료학` `치료법` 제외) | "관찰·훈련" |
| `병리적` | "강한", "심한" |
| `정신병리` | "심리적 어려움" |
| `발병` | "처음 생기는 시점" |

라인 내 부정 명제(`아닙니다`, `아니다`, `대체하지`, `대체하지 않`) 동반 시 자동 면제 (법적 면책 의도).

### 1.4 메타 회피 칭찬 (META_AVOIDANCE_PATTERNS) — 3 패턴

> **2026-05-10 파운더 자기분석 review §4.5 도출.** CBT 도구를 학습한 사용자가 "나는 ○○를 정확히 인식했다"를 자기 정당화 자료로 쓰는 회피 패턴을 카피 단계에서 차단.

| 금지 어휘 | 왜 |
|---|---|
| `당신은/이 정확히/완벽하게/훌륭하게/이미 인식/이해/파악/발견/분석` | 사용자가 메타인지 성공을 행동 회피의 정당화로 사용할 위험 |
| `훌륭한/하게/히 인식/분석/성찰/통찰/발견` | 동일 |
| `완벽한/하게/히 인식/분석/파악/통찰` | 동일 |

→ 사용자의 메타인지 성공은 자체로 가치가 아니라, **관찰 가능한 행동 변화로 이어졌을 때만** 의미가 있다 (운영 원칙 #1·#3 정합).

## 2. 권장 어휘 (분석가 톤 강화)

### 2.1 동사 — 행동 동사 (자율성 지향)

| 권장 | 회피 |
|---|---|
| 관찰하세요 / 기록하세요 | 함께 들여다봐요 |
| 측정하세요 / 비교하세요 | 느껴보세요 |
| 분석하세요 / 분류하세요 | 마음을 들여다보세요 |
| 점검하세요 / 확인하세요 | 한번 살펴봐요 |

### 2.2 명사 — 도구·데이터 명사

| 권장 | 회피 |
|---|---|
| 데이터, 패턴, 신호, 지표 | 마음, 감정, 느낌 (서비스 본질 아님) |
| 사고, 생각, 자동 사고 | 마음의 소리, 내면 |
| 왜곡, 분류, 카테고리 | 부정적 마음 |
| 출력, 기록, 시계열 | 흔적 |
| 준거점, 변동성, 민감도 | 상태 |

### 2.3 정량 표현 (데이터 우선)

| 권장 형태 | 예시 |
|---|---|
| 숫자·시간·비교 | "5일 연속", "지난 7일과 비교", "1분 소요" |
| 카테고리 | "직장 × 파국화 3회", "관계 × 점쟁이오류 2회" |
| 변화량 | "Δpain -2.3", "autonomy +5" |

### 2.4 톤 마커 — 분석 보고서 어조

| 권장 | 회피 |
|---|---|
| "관찰됐습니다" | "있어요" (응원체 인접) |
| "기록이 누적됩니다" | "쌓여가요" |
| "패턴이 식별됐습니다" | "보였어요" |
| "비교 가능한 데이터가 됩니다" | "도움이 될 거예요" |

## 3. 채널별 적용 가이드

### 3.1 앱 내 카피 (UI text)

- 헤딩: 명사·짧은 문장. 정량 정보 우선
  - ✓ "지난 7일 패턴 — 직장 × 파국화 3회"
  - ❌ "최근 일주일 동안 힘드셨던 순간을 살펴봐요"
- 마이크로카피: 동사 위주, 명령형. "관찰하세요" "기록하세요"
- 토스트·알림: 객관 사실 + 다음 동작
  - ✓ "체크인 저장됐어요. 21시에 패턴 분석 알림이 옵니다."

### 3.2 이메일 (recruitment·follow-up)

- 제목: 정량 또는 사실. "30명 베타 — 14일 데이터 누적 시작"
- 본문: 분석 보고서 어조. "본 베타는 ... 측정합니다" / "참여자에게 ...가 제공됩니다"
- 서명: 운영자 명·역할만. 감정 표현 ❌

### 3.3 푸시 알림 (Web Push)

`lib/notifications/copy.ts` 단일 출처. 본 lexicon 적용 예시:
- ✓ `오늘 체크인 안 됐어요 / 1분이면 한 줄 남길 수 있어요. 지난 7일과 비교 데이터가 쌓입니다.`

### 3.4 소셜·블로그 (CMO Stage 1+)

- 1인칭 ❌. "운영자"·"BlueBird"로 지칭
- 사례 인용: 익명·정량화. "30대 직장인 A씨, 직장 × 파국화 5회 → Δpain -2.1"
- 인용 출처: 학술 원문 (`Aldao 2010`, `Charpentier 2017` 등)

## 4. 카피 작성 체크리스트 (PR 머지 전 사용)

작성자는 다음 5개 질문에 "예"로 답할 수 있어야 한다:

1. **데이터 우선?** — 객관 사실·숫자·카테고리가 있는가? (감정 형용사로 시작하지 않는가?)
2. **감정 배제?** — 위로·응원·공감 단어가 없는가?
3. **자율성 지향?** — 정답을 주는가, 관찰·질문을 유도하는가?
4. **금지 어휘 0?** — `npm run lint:copy` 통과하는가?
5. **분석가 톤 마커 1+?** — "관찰", "기록", "분석", "측정", "패턴", "신호", "지표", "데이터" 중 1개 이상 사용?

5개 모두 "예" — 머지 가능. 1개라도 "아니오" — 재작성.
