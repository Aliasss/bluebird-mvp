# 자발 언급 코딩 rubric — v0.1 (IM.1 Quick-start)

> **소유:** strategy-manager (CSO 산하). **출처:** 5/15 weekly all-hands §5 #F-1 (A-3 추적).
> **목적:** IM.1 베타(30명 × 14일)에 즉시 적용 가능한 1-page 적용판. **v1.0 풀 버전(961 lines)**은 `docs/im1/coding-rubric-v1-2026-05-04.md` 참조 — 본 v0.1은 그 subset.
>
> **차이:** v1.0 29 카테고리 → v0.1 핵심 10 카테고리만. Cohen's κ ≥0.7 가드 동일.
> **사용 흐름:** 코더가 응답 1건 읽음 → 본 1-page 보고 분류 → 의문 시 v1.0 §해당 카테고리 참조.

## 0. 코딩 원칙 (v1.0 §0과 동일, 절대 기준)

1. **유도 어휘 박지 않은 응답만 자발 언급 인정** — 우리 surface 어휘를 그대로 반복하는 응답은 카운트 ❌
2. **동의어 군집 인정** — v1.0 §부록 E 매핑 표 기준. 매핑 사유 1줄 의무 기록
3. **부정 명제도 자발 인정** — "위로받는 건 안 맞아요" = A1 정합 1점
4. **불확실 응답은 경계 case 큐 누적** — 주 1회 strategy-manager + CPO 검토
5. **본질 위협 자발 언급 즉시 escalation** — D 영역 1건 등장 시 당일 CSO + risk-manager 보고
6. **응답 위치 추적** — 질문 ID + 어휘 거리 코딩 노트 기록. surface 어휘로부터 ≤3 어절 내 등장 시 유도 의심 표지

## 2. 카테고리별 1-line 정합 기준 (정합 시 1점, 경계 0.5점, 미정합 0점)

### A1. 감정 언어 부담
- **정합:** "감정 그대로보다 정리하고 싶다" / "기분보다 *왜*가 궁금하다" / "감정 일기 안 맞다" / "위로받는 건 안 맞다"(부정 명제)
- **미정합:** "위로받는 게 필요해요" / "마음이 따뜻해지는 글이 좋아요"
- **경계:** "감정도 분석해보고 싶어요" (분석 의도 ✓, 감정 부담 시그널 약함)

### A2. 분석적 자기이해 선호
- **정합:** "왜 그런 반응을 했는지 알고 싶다" / "내 사고 흐름이 어떤지 보고 싶다" / "근거를 따져보고 싶다"
- **미정합:** "마음을 편하게 하고 싶다" / "그냥 위로받고 싶다"

### B1. "사고를 도구로 다룬다"
- **정합:** "사고를 다룬다" / "사고를 정리한다" / "관찰 도구" / 동의어 군집(시스템·로직·구조·메커니즘) ✓
- **미정합:** "마음을 다스린다" / "감정을 관리한다" (감정 어휘는 B1 X — A1 후보)

### B2. "패턴이 보인다"
- **정합:** "반복되는 패턴" / "비슷한 상황에서 똑같은 반응" / 동의어(루틴·습관·구조) ✓
- **미정합:** "그날그날 달라요"

### C1. 직장 트리거
- **정합:** "회사" / "업무" / "상사" / "동료" / "발표" / "회의" / "성과 평가" 어휘 등장
- **미정합:** 직장 외 영역만 언급

### C2. 관계 트리거
- **정합:** "친구" / "가족" / "연인" / "부모" / "관계" 어휘 등장
- **미정합:** 관계 외 영역만 언급

### D1. "위로받고 싶다" 명시 (본질 위협 #1)
- **정합:** "BlueBird가 따뜻하게 위로해줬으면" / "감성적인 메시지가 좋겠다" / "공감해주는 챗봇이면 좋겠다"
- **escalation:** ≥10% 누적 시 카피 hotfix 트리거 (v1.0 §부록 F)

### D4. "친구처럼" 명시 (본질 위협 #4)
- **정합:** "친구 같은 챗봇" / "캐릭터가 귀여웠으면" / "말투가 친근했으면"
- **escalation:** 동일

### E1. 디스턴싱 비교 언급
- **정합:** "디스턴싱 써봤는데" / "디스턴싱이랑 비교하면" / "디스턴싱은 ○○인데 BlueBird는"
- **추가 기록:** 비교 차이 항목 (톤·자기상·개입·시간·가격·메타포 중 하나)

### F1. "돈 낼 의향" 자발 언급
- **정합:** "돈 내고 쓸 것 같다" / "월 ○○원 정도면" / "유료라면 ○○ 정도까지"
- **미정합:** "무료라서 좋아요" / "유료면 안 쓸 것 같아요" (단 결제 의향 ≥20% 게이트 측정에는 negative로 카운트)

## 3. Cohen's κ 가드 (v1.0과 동일)

- **임계:** ≥ 0.7
- **측정 시점:** 첫 5명 응답 batch 완료 후
- **미달 시:** v0.2로 재정의 (정합 기준 명시화). 또는 v1.0의 해당 카테고리 정의로 escalate
- **코더:** 최소 2명 (strategy-manager + CPO 또는 senior-ux-researcher)

---

## 4. 코딩 워크플로우 (15-min loop)

1. 응답 1건 읽음 (스크리닝 or 인터뷰)
2. 본 §2의 10개 카테고리 각각에 대해 1점/0.5점/0점 부여 (체크리스트)
3. 경계 case → §부록 D 큐 추가
4. D1·D4 정합 시 → 당일 escalation 메시지 작성
5. E1 정합 시 → 비교 차이 항목 추가 기록
6. 코딩 노트 형식:
   ```
   응답ID: R-001
   카테고리: A1=1, A2=1, B1=0, B2=0.5, C1=1, C2=0, D1=0, D4=0, E1=0, F1=0
   surface 어휘 거리 체크: clean (직접 surface 어휘 없음)
   동의어 매핑: B2 "루틴" → "패턴" 군집 (v1.0 §부록 E)
   특이사항: 디스턴싱 언급 후보 — 명시적 비교 부재라 E1 0점 처리
   ```

## 5. 결과 산출

IM.1 D14 시점에 strategy-manager + CPO + data-analyst가 다음 산출:

| 지표 | 산식 |
|---|---|
| 자발 가치 언급 비율 | (B1=1 OR B2=1)인 응답 수 / 전체 응답 수 |
| 결제 의향 비율 | F1=1인 응답 수 / 전체 응답 수 |
| 본질 위협 비율 | D1=1 OR D4=1인 응답 수 / 전체 응답 수 |
| 1차 타겟 적중률 | A1+A2 평균점 ≥1.5인 응답 비율 |
| 디스턴싱 비교 인지율 | E1=1인 응답 비율 |

→ PMF 게이트 3종(자발 ≥30% · 결제 의향 ≥20% · 30일 잔존율 ≥15%) 중 2종을 본 rubric으로 직접 산출.
