## 0. 코딩 원칙 (CSO·CPO 합의 사전 반영)

본 rubric은 다음 6개 원칙을 절대 기준으로 한다. 어느 카테고리 코딩이든 이 원칙과 충돌 시 본 §0 우선.

1. **유도 어휘 박지 않은 응답만 자발 언급으로 인정** — 모집 공고·스크리닝 폼·인터뷰 가이드에서 우리가 surface한 어휘는 자발 카운트 X. 명시적 surface 어휘 목록:
   - 카테고리 정의 어휘: "사고를 디버깅하는 도구"·"디버깅"
   - 1단락 보조 어휘: "시스템처럼"·"패턴"·"사용설명서"
   - 인터뷰 가이드 surface (M30.1): "분석적 vs 정서적"·"디스턴싱·마인드카페·하루콩"·"분석직·창의직·관리직·서비스직"
   - 응답에 동일 표현 등장 시 자발 X. *질문에 박혀 있던 어휘를 반복하는 것*은 카운트 X.

2. **응답 어휘가 우리 키워드의 *동의어*면 자발로 인정** — 의미 매핑 표 (§부록 E)에 따라 동의어 군집 인정.
   - 예: "프로그램·시스템·로직·구조·메커니즘·작동 방식·체계" → "구조"의 동의어 군집
   - 예: "디버그·점검·고친다·튜닝·최적화" → "디버깅"의 동의어 군집
   - 예: "OS·운영체제·플랫폼·엔진" → 시스템 메타포 동의어 군집
   - 단, 동의어 인정 시 *코딩 노트에 원문 인용 + 매핑 사유 1줄* 의무.

3. **부정 명제도 자발로 인정** — 응답이 *우리가 측정하려는 가치를 명시적으로 거부*하는 형태도 동일 카테고리로 카운트.
   - 예: "그건 위로 받는 거지 저한테 안 맞아요" → A1 (감정 언어 부담) 정합 1점
   - 예: "감정 일기 같은 건 못 쓰겠어요" → A1 정합 1점
   - 예: "친근한 캐릭터는 부담스러워요" → D2 본질 위협 *반대* 시그널 (1차 타겟 적중 보강)

4. **불확실 응답은 별도 분류 — 경계 case 큐 누적** — 두 코딩자가 *동시에 유보*한 응답은 §부록 D에 누적. 누적된 사례는 주 1회 strategy-manager + CPO 검토 회의 (15분)에서 결정.

5. **본질 위협 자발 언급은 즉시 escalation** — D 영역 6개 위협 중 1건 자발 등장 시 카운트 동시에 *당일* CSO + risk-manager에게 보고. ≥10% 누적 시 카피 hotfix 트리거 (§부록 F).

6. **자발성 가드 — 응답 위치 추적** — 응답이 어느 질문에 대한 답인지 *질문 ID + 어휘 거리*를 코딩 노트에 기록. 우리 surface 어휘로부터 ≤ 3 어절 내 등장 시 *유도 의심 case*로 별도 표지.

### A. 1차 타겟 3축 (selectee 통과 결정)

#### A1. 심리·동기 (감정 언어 부담)

- **측정 의도**: positioning §2 1차 타겟 3축 중 *심리·동기* 축. 응답에서 감정 언어를 *불편해하거나 · 우회하거나 · 시스템화*하는 시그널.
- **입력 위치**: 스크리닝 Q1 (마음에 걸리는 일을 어떻게 다루는지) + 인터뷰 도입부.
- **정합 sample (5)**:
  1. "감정을 그대로 표현하기보다는 정리하고 싶어요"
  2. "마음이 어땠는지보다 *왜* 그랬는지가 더 궁금해요"
  3. "내가 어떤 패턴인지 보고 싶어요" *(질문에 "패턴" 어휘 없을 때만 자발 인정 — §0 #1 가드)*
  4. "감정 일기 같은 건 안 맞더라고요"
  5. "기분이라고 적기보다 상황·반응을 적게 돼요"
- **미정합 sample (3)**:
  1. "마음이 따뜻해지는 글이 좋아요"
  2. "감정을 그대로 받아들이는 게 중요하죠"
  3. "위로받는 게 필요해요"
- **경계 case (2)**:
  - "감정도 분석해보고 싶어요" — 분석 의도가 있으나 감정 자체를 부담으로 보지 않음 → **0.5점**
  - "그냥 적어두고 잊어요" — 우회는 있으나 시스템화 의도 X → **0.5점**
- **분류**: 0 (미정합) / 0.5 (경계) / 1 (정합)
- **§0 #3 부정 명제 인정**: "위로받는 건 저한테 안 맞아요" → 정합 1점

#### A2. 인지 스타일 (분석적 자기이해 선호)

- **측정 의도**: positioning §2 *인지 스타일* 축. "위로받기"보다 "구조 보기"를 선호하는 자기상.
- **입력 위치**: 스크리닝 Q2 (본인을 알아가는 도구·방법) + 인터뷰 "BlueBird를 친구에게 어떻게 설명".
- **정합 sample (5)**:
  1. "내 사고에서 반복되는 흐름을 짚어보는 게 좋아요"
  2. "어떤 책을 읽으면서 내 행동의 메커니즘을 알게 됐을 때가 가장 도움이 됐어요"
  3. "노션·sheets에 정리하면서 패턴이 보였을 때 의미 있었어요"
  4. "MBTI는 좀 단순했는데, 그래도 *유형으로 보는 시도*는 도움 됐어요"
  5. "내 머릿속이 어떻게 돌아가는지 다이어그램처럼 보고 싶어요"
- **미정합 sample (3)**:
  1. "친구랑 이야기하면 풀려요"
  2. "그냥 시간이 약이에요"
  3. "그런 도구는 안 써봤고 별로 필요도 없어요"
- **경계 case (2)**:
  - "심리 검사는 해봤는데 별 효과 없었어요" — 구조화 도구 경험은 있으나 평가가 부정적 → **0.5점**
  - "걷기·명상이 도움 됐어요" — 비구조화 방법이 우세 → **0점** (단 추가 진술에 구조화 도구 등장 시 재평가)
- **분류**: 0 / 0.5 / 1
- **§0 #2 동의어 인정**: "시스템·로직·메커니즘·구조·체계" 자발 등장 시 정합 1점.

#### A3. 맥락 (한국 인지노동자)

- **측정 의도**: positioning §2 *맥락* 축. 직장·체면·자기낙인 맥락 자발 surface.
- **입력 위치**: 스크리닝 Q3 (생각을 많이 쓰는 시간 + 맥락) + 인터뷰 차별화 질문.
- **정합 sample (5)**:
  1. "회의가 끝나고도 머릿속에서 계속 굴러가요" (인지노동 동작 + 시간 함의)
  2. "문서를 많이 쓰는데 끝나면 머리가 비어요"
  3. "결정 내릴 일이 많은 하루였고, 그 결정 이후에도 곱씹는 시간이 길어요"
  4. "리뷰·피드백 받는 자리가 많아서 그 후에 자기 점검 시간이 깁니다"
  5. "고객·동료한테 보이는 모습 신경 쓰여서 늘 한 번 더 검토해요" (체면 맥락 자발)
- **미정합 sample (3)**:
  1. "생각 별로 안 해요"
  2. "몸 쓰는 일이라 머리는 비어 있어요"
  3. "학생이라 생각 시간은 공부할 때만요"
- **경계 case (2)**:
  - "하루 6시간 컴퓨터 앞에 있어요" — 시간만 명시 · 인지노동 동작 모호 → **0.5점**
  - "프리랜서라 자기 검토 시간이 많아요" — 인지노동 동작 + 자기 검토 자발 등장 → **1점**
- **분류**: 0 / 0.5 / 1
- **가드**: 직무 narrowing 어휘 (BA·PM·엔지니어·컨설턴트·연구자) 자발 등장 시 *코딩 노트에 분포 기록만*. 통과 결정에는 직무 분포 사용 X. positioning §2 정정 (직무 enumerate ≠ 한정).

#### A 영역 통과 조건 (selectee 결정)

- **3축 ALL 정합 (Q1+Q2+Q3 각 1점)**: 1순위 selectee — 무조건 인터뷰
- **2축 정합 + 1축 0.5점 (합계 ≥ 2.5)**: 2순위 selectee — 정원 미달 시 인터뷰
- **1축 이하 또는 합계 < 2.5**: 미통과 — positioning §2 2순위 확장 후보 추후 활용 동의 묻기 (스크리닝 Q5 동의 항목에 포함됨)

목표: selectee 30명 중 ≥ 60% 가 3축 ALL 정합 (recruitment-package §0 모집 목표).

### B. PMF 핵심 가치 자발 언급 (PMF plan §0 ≥30% 게이트)

#### B1. 카테고리 정합 — "사고 디버깅" 자발 언급

- **측정 의도**: PMF plan §3 M30.1 자발 가치 언급. 응답에서 *우리가 안 박은* 카테고리 어휘. positioning §1 카테고리 정의("사고를 디버깅하는 도구")가 사용자 언어로 자발 등장하는가.
- **입력 위치**: 인터뷰 "BlueBird를 친구에게 어떻게 설명" · "다른 정신건강 앱과 어떻게 다른가요".
- **정합 sample (5)**:
  1. "내 사고를 *디버깅*하는 느낌"
  2. "생각의 패턴을 보는 도구"
  3. "OS 같다 — 내 운영체제를 보는"
  4. "진단해주는 것 같아요. 진료가 아니라 진단"
  5. "내 머릿속을 *튜닝*하는 도구"
- **미정합 sample (3)**:
  1. "마음 챙김 앱이에요"
  2. "스트레스 관리에 좋은 앱"
  3. "심리 상담 대신 쓰는 앱"
- **경계 case (2)**:
  - 모집 공고에 "디버깅" 어휘가 박혔으므로 응답에 "디버깅"이 단순 등장하면 자발 X. *사용자가 자기 언어로 풀어 설명*했을 때만 정합 (예: "사고를 *고치는*"·"점검하는"·"문제를 짚어내는"). → 코딩 노트에 원문 인용 의무.
  - "이상한 생각 잡아내는 앱" — 카테고리 동의어 군집 매핑 적용 → **1점** (§부록 E)
- **분류**: 0 (미정합) / 1 (정합)
- **PMF 게이트 기여**: B1 + B2 + B3 + B4 합산. ≥ 30% 응답자에서 *최소 1개* 카테고리 자발 언급 시 PMF plan §0 임계 통과.

#### B2. 분석가 톤 (분석적·구조적 인지)

- **측정 의도**: positioning §4 차별화 3축 중 *톤* 축. 사용자가 BlueBird를 분석적·구조적 도구로 인식하는가.
- **입력 위치**: 인터뷰 "분석적 vs 정서적 어느 쪽" 질문에 *답하기 전* 다른 답변에서 자발 surface하는지가 핵심. *질문에 박힌 어휘 반복*은 자발 X.
- **정합 sample (5)**:
  1. "위로받는 도구가 아니라 *짚어주는* 도구로 느껴져요"
  2. "내 사고를 *데이터처럼* 보게 돼요"
  3. "객관적으로 *제3자 시점*에서 본인을 보는 느낌"
  4. "분석가 옆에서 같이 검토하는 느낌"
  5. "*반증*을 던져주는 게 좋아요" (§5 이론 백본 Heuer ACH 정합)
- **미정합 sample (3)**:
  1. "따뜻해서 좋아요"
  2. "마음이 편해져요"
  3. "친구처럼 위로받는 느낌"
- **경계 case (2)**:
  - "정확한데 좀 차가워요" — 분석가 톤 인식 + 부정적 평가 → **1점 (인식)** + 코딩 노트에 *부정 평가* 별도 표지
  - "분석적이면서 따뜻해요" — 하이브리드 인식 → **0.5점** (PMF plan §11.4 의문 제기 입력)
- **분류**: 0 / 0.5 / 1
- **트리거 연결**: 본 카테고리 *반대* 시그널(정서적 우세) → C5 트리거.

#### B3. 한국어 fine-tuning (체면·자기낙인 어휘)

- **측정 의도**: positioning §3 차별 가치 *한국어 fine-tuning*. 사용자가 글로벌 도구·번역 도구와 다른 한국어 정밀도·문화 정합을 자발 인지하는가.
- **입력 위치**: 인터뷰 "다른 도구와의 차이" + "체면·자기낙인 맥락" 자유 답변.
- **정합 sample (5)**:
  1. "한국 사람 머릿속을 알고 짠 것 같아요"
  2. "체면 챙기는 표현이 나올 때 그걸 *알아채는* 게 신기했어요"
  3. "*돌려 말한* 표현을 풀어서 보여주는 게 좋아요"
  4. "Wysa·Replika 한국어는 어색해서 못 썼는데 이건 다르네요"
  5. "직장인 톤을 알아요. 영어권 앱은 그게 안 돼요"
- **미정합 sample (3)**:
  1. "한국어니까 편해요" (단순 언어 적응 — fine-tuning 인식 X)
  2. "번역체가 아니라서 좋아요" (단순 자연어 — 정밀도 X)
  3. "한국어 지원 앱이 별로 없죠" (인프라 인식만)

  4. "이 데이터가 *내 자산* 같다는 생각이 들었어요" (손실 회피 시그널 — H1·H2 사전)

  - "기록은 남기는데 다시 안 봐요" — 누적 인식은 있으나 가치 인식 X → **0.5점**
  - "내가 정말 그런 사람인지 *결정짓고 싶진 않은데* 흥미로워요" — H1 가드 (b)·(c) 정합 시그널 → **1점** + 코딩 노트에 가드 정합 표지

- **트리거 연결**: 매뉴얼 자발 ≥10% → 매뉴얼 prototype 격상 (PMF plan §11.3) — C1과 동일 측정 데이터 공유.

#### B 영역 PMF 게이트 산출 공식

`B_score(응답자) = 1 if (B1=1 OR B2≥1 OR B3≥1 OR B4≥1) else 0`

PMF plan §0 자발 언급 임계 = `Σ B_score / N ≥ 0.30` (N=30 응답자 누적 시점에 평가).

부분 정합(0.5)은 게이트 산출 공식에 미포함. 단 §부록 D 경계 case 큐에 누적해 v1.1에서 0.5의 게이트 가중 여부 재정의.

## 부록 C. Cohen's κ 측정 SOP

### C.1 입력 매트릭스

- 코딩 매트릭스: **2명 × 5 응답자 × 27 카테고리 = 270 cell** (첫 batch)
- 각 cell 값: 카테고리별 분류 기준 (0 / 0.5 / 1 또는 다중 분류)

### C.2 κ 계산식

`κ = (P_o - P_e) / (1 - P_e)`

- `P_o` = 관찰된 일치 비율 = 두 코딩자 점수 동일 cell / 전체 cell
- `P_e` = 우연 일치 비율 = Σ (P_코딩자1_class × P_코딩자2_class)

### C.3 도구

- 권장: scikit-learn `cohen_kappa_score(y1, y2)` (Python)
- 대안: 수동 계산 (Google Sheets COUNTIF + 산식)
- 다중 분류 (C4·C5·C6·C7·C8·C9·E1·E2·E3 등)는 *각 분류를 0/1 binary화* 후 카테고리별 κ 산출

### C.4 95% 신뢰구간 (N≥30 시)

`SE = sqrt(P_o × (1 - P_o) / N)` (정규근사)

`95% CI = κ ± 1.96 × SE / (1 - P_e)`

### C.5 κ 해석

| κ 범위 | 해석 |
|---|---|
| 0.0 ~ 0.20 | poor |
| 0.21 ~ 0.40 | fair |
| 0.41 ~ 0.60 | moderate |
| 0.61 ~ 0.80 | substantial (**가드 ≥ 0.7**) |
| 0.81 ~ 1.00 | almost perfect |

회의록 §4.3 합의: 첫 5명 후 κ ≥ 0.7 미달 시 rubric v1.1 재정의.

### C.6 카테고리별 vs 종합 κ

- **종합 κ**: 270 cell ALL에 대한 1개 값
- **카테고리별 κ**: 각 27 카테고리 × 10 cell (5응답×2코딩자) — 어느 카테고리가 약한지 진단

종합 κ ≥ 0.7 + 카테고리별 κ ALL ≥ 0.5 = v1.0 유지. 종합 OK이나 카테고리 1개 < 0.5 = 해당 카테고리만 v1.1 재정의.
