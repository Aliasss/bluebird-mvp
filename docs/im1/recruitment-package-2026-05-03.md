## 1. 모집 채널 (옵션 + 권고)

각 채널마다 4축 평가: 채널 특성(도달·인구) / 1차 타겟 3축 적합도 / 게시 톤 / 위험·권고.

채널 후보 6개 모두 분석. 채택은 CEO 결정 (본 패키지 §7 #1).

### 1.1 LinkedIn 한국 인지노동자 그룹

- **채널 특성**: 한국 LinkedIn 활성 사용자 추정 ~300만. 직무·경력 자기 표시가 일상화된 환경. 게시물 알고리즘이 텍스트 길이·전문성 시그널을 우대.
- **1차 타겟 3축 적합도**:
  - 심리·동기 축: △ (감정 언어 부담은 평균 수준 — LinkedIn은 "성취" 톤이 우세, "사고를 시스템처럼 본다" 자기상에 부분 적중)
  - 인지 스타일 축: ○ (분석적 자기이해 선호자가 비례 이상 분포 추정)
  - 맥락 축: ◎ (한국 인지노동자 dense — 직장 맥락이 강함)
- **게시 톤**: 공식·전문 (장문 OK, 근거·메타포 OK). 장문 투고는 노출 가중.
- **위험**:
  - 직장 노출 위험 — "내 정신 상태를 분석한다"는 자기 표시는 일부 응모자에게 체면 부담
  - 셀프 브랜딩 톤이 우세한 채널 — 응모자 자기보고 편향 가능 ("나는 분석적이다" 자처)
- **권고**: ◎ 1순위. 한국 인지노동자 dense + 분석가 톤 정합. 단 "내 회사 동료가 본다" 위험 완화 위해 응모는 외부 폼(Tally / Google Forms 등)으로 분리.

### 1.2 Threads (Meta) — 텍스트 중심 한국 사용자

- **채널 특성**: 한국 활성 사용자 ~수백만 추정. Twitter 대체 수요 흡수, 텍스트 위주, 알고리즘이 *짧은 호흡 + 호기심 자극*을 우대.
- **1차 타겟 3축 적합도**:
  - 심리·동기 축: △ (감정 표현 자유로운 채널 — 일부 사용자는 정서적, 일부는 분석적)
  - 인지 스타일 축: ○ (분석적 자기이해 선호 사용자 일부 dense — 자기성찰 글이 트래픽을 받음)
  - 맥락 축: ○ (한국 인지노동자 비율 추정 적중)
- **게시 톤**: 캐주얼·짧은 호흡 (장문 분리 게시 권장). 메타포는 OK이지만 격식은 LinkedIn보다 낮음.
- **위험**:
  - 단문 톤 강제 → 카테고리 정의가 명료히 전달되지 않을 위험
  - 알고리즘이 자극·논쟁을 우대 — 분석가 톤이 도달 약할 가능성
- **권고**: ○ 2순위. 도달 보조용. *반드시* LinkedIn 본문과 동일한 카테고리 정의 1줄 ("사고를 디버깅하는 도구") 박을 것.

### 1.3 Disquiet (한국 메이커 커뮤니티)

- **채널 특성**: 한국 IT·스타트업·메이커 커뮤니티. 활성 사용자는 작지만 dense. 분석적 자기이해 선호 사용자 비율이 *비례 이상* 추정.
- **1차 타겟 3축 적합도**:
  - 심리·동기 축: ○ (감정 언어 부담 — 메이커 사용자에게 빈번한 자기상)
  - 인지 스타일 축: ◎ (분석적 자기이해 dense — "구조 보기" 자기상 자연)
  - 맥락 축: ◎ (한국 인지노동자 + 자기 운영 관점 dense)
- **게시 톤**: 공식·간결·기술적 OK. "메이커가 만든 도구"로 보일 위험에 주의.
- **위험**:
  - 도달 absolute 작음 — 응모 1주 < 10명 위험
  - 베타 테스트 권유가 *흔한* 채널 → 응모 피로 가능
  - "스타트업 자기 홍보"로 인식되면 지속 신뢰 ↓
- **권고**: ◎ 1순위 (dense). 단 도달 작으므로 LinkedIn과 *조합*. 이 채널 단독으로 30명 모집 X.

### 1.4 사내 네트워크 / 1촌 추천

- **채널 특성**: CEO·동료의 1촌 네트워크. 도달 작지만 신뢰·진정성 ↑. *시드 응모자* 확보용.
- **1차 타겟 3축 적합도**:
  - 심리·동기 축: 의존 (CEO 네트워크 구성 분포에 따름)
  - 인지 스타일 축: 의존
  - 맥락 축: ◎ (한국 인지노동자 — CEO 동종 직군 추정)
- **게시 톤**: 1:1 직접 메시지 (DM·이메일·메신저). 게시물 X.
- **위험**:
  - 사회적 압력 응모 — "거절하기 어려워서" 응모할 위험 (3축 적중률 ↓)
  - 데이터 정직성 ↓ — 인터뷰에서 *예의상* 답변 편향
  - 표본 편향 — CEO 네트워크 동질성 (직군·세대) 한쪽 쏠림
- **권고**: △ 보조. 30명 중 ≤ 5명만 (≤ 17%). 응모 시 "사회적 압력 없이 자율 거절 가능" 명시.

### 1.5 직장인 커뮤니티 (블라인드·리멤버 등)

- **채널 특성**: 한국 직장인 dense. 익명성 ↑. 자기 노출 부담 ↓.
- **1차 타겟 3축 적합도**:
  - 심리·동기 축: △ (직장인 일반은 "위로받기" vs "구조 보기" 분포 ~ 6:4 추정 — 1차 타겟 dense 아님)
  - 인지 스타일 축: △ (분포 평균 — dense 채널 아님)
  - 맥락 축: ◎ (한국 직장인 dense)
- **게시 톤**: 익명·짧음·자극적. 분석가 톤은 채널 톤과 충돌.
- **위험**:
  - **본질 위협 #3** — "스트레스 관리" 입구로 흐를 강력한 압력. 채널 톤이 정서·고충 토로 우세.
  - 응모자 3축 적중률 ↓ 추정 (응모율 ↑이지만 selectee 통과율 ↓)
  - 베타 테스터 모집 게시는 광고로 분류되어 삭제 위험
- **권고**: ✗ 비권고. 채널 톤이 BlueBird 카테고리 정의와 충돌. 진입 시 본질 위협 신호 #3 위험 ↑.

### 1.6 Twitter/X — 분석적 사고 관심층

- **채널 특성**: 한국 활성 사용자 감소 추세이나 분석가·기술·연구 niche dense. 알고리즘이 논쟁·인용을 우대.
- **1차 타겟 3축 적합도**:
  - 심리·동기 축: ○ (자기상 표시가 자유로운 채널 — "시스템처럼 본다"는 자기상 dense 일부)
  - 인지 스타일 축: ◎ (분석적 자기이해 선호 사용자가 비례 이상 추정 — 한국 분석가·연구·기술 niche)
  - 맥락 축: ○ (한국 인지노동자 비율 적중)
- **게시 톤**: 짧음·인용 가능·메타포 OK.
- **위험**:
  - 도달 알고리즘이 *자극*을 우대 — 분석가 톤이 약하게 퍼질 위험
  - 한국 활성 사용자 감소 추세 — 도달 absolute 추정 ↓
  - 인용 RT시 카피 일부만 노출 → 카테고리 정의 손실 가능
- **권고**: ○ 2순위. 도달 보조용. 알고리즘 영향 큼 — 자체 도달 변동성 ↑.

### 1.7 CPO 권고 — 채널 우선순위 + 조합

**1순위 조합 (권고)**: LinkedIn (1.1) + Disquiet (1.3) **동시** 게시.
- 근거 (의사결정 우선순위 적용):
  - **차별화 정합성** (CPO 우선순위 #2): 두 채널 모두 분석가 톤과 정합. 정서적 채널(블라인드·인사이드 류) 회피.
  - **Falsifiability** (CPO 우선순위 #1): LinkedIn은 도달·맥락 dense, Disquiet는 1차 타겟 적중률 dense — 둘이 상호보완 (도달 vs 적중률 trade-off 동시 해소).
  - **사용자 인지 부하** (CPO 우선순위 #4): 두 채널 모두 응모 절차가 외부 폼으로 통일 가능 (응모자 부하 일정).

**2순위 보조** (1순위 응모 < 10명/주 시 활성화): Threads (1.2) + Twitter/X (1.6) 추가. *동일 카피*로.

**보조 라인**: 1촌 추천 (1.4) — 30명 중 ≤ 5명 (≤ 17%) 한정. 사회적 압력 가드 명시.

**비권고**: 직장인 커뮤니티 (1.5) — 본질 위협 #3 위험.

## 3. selectee 사전 스크리닝 폼 — 5문항

원칙:
- 직무 묻지 않음 (직무 narrowing 0)
- *유도하지 않는* 자유 응답 형식 (객관식 X — 정답 시그널 회피)
- 정답이 보이지 않는 질문 (3축 직접 어휘 surface 0)

- *유도 어휘* "분석적·구조·디버깅·자율성·시스템·패턴·인지 왜곡" 박지 않음

### 문항 1 — 심리·동기 축 (감정 언어 부담)

**질문 (사용자 노출 카피)**:
> Q1. 최근 본인이 *마음에 걸리는 일*을 어떻게 다루시는지 자유롭게 적어주세요. (3~5문장)

**측정 의도**: 1차 타겟 3축 중 *심리·동기* 축. 응답에서 자기를 *시스템처럼* 다루는 자기상이 자발 등장하는가.

**유도 어휘 surface 자체 점검**:
- "분석적" surface 0 ✓
- "구조" surface 0 ✓
- "시스템" surface 0 ✓
- "감정" surface 0 (감정 언어 부담을 *측정*하므로 박지 않음) ✓

**코딩 rubric 입력 (strategy-manager 5/4 합의 예정)**:
- 정합 (3축 중 1축 적중): 답변에 "정리"·"펴본다"·"이유 찾는다"·"왜 그렇게 됐는지"·"단계별로" 같은 *분석 동작* 자발 등장
- 부분 정합: 답변이 "기록한다"·"잠시 둔다" 등 중립 동작
- 부적합: 답변이 "위로받는다"·"잊으려 한다"·"혼자 견딘다" 등 정서·회피 동작 우세

**통과 조건 기여**: 정합 시 3축 중 1점.

**측정 의도**: 인지 스타일 축. "위로받기"보다 "구조 보기"를 선호하는 자기상이 자발 등장하는가.

**유도 어휘 surface 자체 점검**:

## 부록 A. 자발 언급 코딩 rubric 협업 사항

스크리닝 폼 5문항의 코딩 rubric은 strategy-manager가 5/4에 합의 예정 (회의록 §5 #5 — 5/4 (월) deadline).

본 패키지 §3의 5문항이 그 rubric의 *입력*이므로:
- **문항 변경 시 rubric 재합의 필요**
- 5/3 본 패키지 → 5/4 rubric 합의 → 5/5 EOD prerequisite 1~6 ALL PASS 보고 → 5/6 (수) 모집 시작 (또는 5/9 deadline)
- 본 패키지 §3 5문항 변경은 strategy-manager·CSO·CPO 3자 합의 필수

또한 인터뷰 단계 자발 언급 코딩(M30.1)은 본 패키지 §3 스크리닝 코딩과 *독립*. 스크리닝은 selectee 통과 게이트, 인터뷰는 PMF 게이트.
