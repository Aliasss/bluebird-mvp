## 0. CEO 발의 (개회 시점)

> "혹시 '콘텐츠 피드'를 도입하는 건 어떨까? 우리가 인지왜곡으로 인해 불안을 느끼는 것에 대해 다양한 이론들을 토대로 친절히 설명해주는 콘텐츠를 생산하고 이를 노출하는 콘텐츠 공간을 서비스 내에 배치하는거지."

**제안 핵심 요소**:
- (a) 별도 *콘텐츠 공간* (서비스 내 신규 카테고리)
- (b) 콘텐츠 주제: 인지 왜곡·불안 메커니즘·학술 이론(CBT·DPT·전망이론·CAS·SDT 등)
- (c) 톤: "친절히 설명"
- (d) 시점: MVP 베타 테스트 *이전* 도입 여부

## 1. 제안 범위 명확화 (CEO 보충)

CEO 추가 설명 없음. 회의 진행 중 파생되는 모호함은 페르소나 검토 시 *각자의 가정으로 분리 surface*하기로 합의.

가정 분리:
- **가정 1**: "별도 공간"은 dashboard 외 신규 메뉴·라우트(예: `/feed`).
- **가정 2**: 콘텐츠는 *정적*(수동 작성·CMS 없음) 시작 가능.
- **가정 3**: "친절히 설명"의 톤·길이는 미정 — 회의에서 결정.
- **가정 4**: 푸시 알림·정기 발행은 미정 — 회의에서 결정.

## 2. 페르소나별 검토

### 2.1 CPO — 명시적 비목표·본질 위협 6 시그널 점검

⚠️ **3축 점검 결과 — 다수 위반 가능성**:

| 비목표·신호 | 콘텐츠 피드와의 충돌 |
|---|---|
| 명시적 비목표: 외부 마케팅 | G3 통과 후. 콘텐츠는 *외부 SEO/유입* 기능을 겸할 수 있어 트리거 없이 surface하면 사전 박는 셈. |
| 본질 위협 #1 정서·치유 톤 | "친절히 설명"은 *코칭 톤·치유 톤*과 거리 가깝다. 분석가 톤 차별화 약화. |
| 본질 위협 #4 정기 자기 라벨링 | 콘텐츠 *정기 발행·푸시*가 들어가면 즉시 위반. |
| 본질 위협 #6 "외롭지 않게" | 콘텐츠로 사용자를 *동반·위로*하려는 의도가 surface되면 위반. |
| 카테고리 정의 | "사고 디버깅 도구" — *콘텐츠 소비 도구*는 다른 카테고리. 사용자 멘탈 모델에 카테고리 표류 위험. |

**CPO 의견**: 본 제안은 *별도 공간 + 정기 노출* 형태이면 **거부**. 단 *콘텐츠 자체의 가치*는 인정 — 학술 근거 surface는 분석가 톤 강화에 정합. **카테고리 표류 없는 형태로의 변환** 필요.

**Falsifiability 점검**: 콘텐츠 피드 도입이 *어떤 가설*을 검증하는가? "콘텐츠 학습 → 디버깅 사이클 완료율 ↑"? 측정 메커니즘 미정. 가설 명시 후 측정 가능한 명제로 변환 필요.

### 2.2 CSO — 디스턴싱·하루콩 비교 + 차별화 3축

🔍 **경쟁 카테고리 정합 점검**:

- 디스턴싱(추정)·하루콩 같은 정신건강 앱 카테고리는 *콘텐츠 피드 흔함* — 추정컨대 명상·웰빙 콘텐츠.
- BlueBird가 *별도 콘텐츠 피드*를 추가하면 **사용자에게 같은 카테고리로 보일 위험** — 차별화 약화.
- 차별화 3축 (`positioning §3.3.1`):
  - 톤: 분석적 vs 정서적 → "친절히 설명"은 톤 약화
  - 자기상: 운영자 vs 회복자 → 콘텐츠 소비자는 *수동* 자기상
  - 메타포: 디버깅·OS·항해 → 미디어·블로그는 메타포 다른 축

**CSO 의견**: 별도 피드는 **차별화 약화** — 거부. 단 *카테고리 표류 없는 형태*에는 동의 가능.

### 2.3 CTO — 구현 비용·인프라 영향

🔧 **공수 추정**:

- 별도 피드 라우트(`/feed`) + 카드 컴포넌트 + 콘텐츠 모델 + 라우팅: 1~2 sprint
- CMS (Notion API 또는 자체) 또는 markdown 파일 기반 정적 빌드: +0.5 sprint
- 사용자 읽음·체류시간 추적: +0.3 sprint (analytics_events 활용)
- 추천·정렬: 베타 단계 미적용 가능 (단순 시간순).
- PWA 캐시 정책 추가, SEO meta tag, 접근성: +0.5 sprint
- **총 ~3 sprint 추가** — IM.1 모집·E2E 검증보다 우선순위 낮음.

**CTO 의견**: PMF 검증 *전*에 신규 카테고리 기능 추가는 **PMF 신호 노이즈를 만든다**. 베타 사용자 30명이 "콘텐츠 좋아요"라고 답해도 그게 *사고 디버깅 도구* 가치 신호인지 *콘텐츠 도구* 신호인지 분리 어려움. 거부.

### 2.4 product-designer — 디자인 토큰·인지 부하·시각 메타포

🎨 **디자인 risk 5종**:

1. **본질 위협 #2 디자인 수렴**: 콘텐츠 카드 = 블로그·미디엄 시각 어휘 → 파스텔·일러스트로 흐를 강한 압력. 디자인 토큰(refined minimalism)과 충돌.
2. **인지 부하 ↑**: dashboard 외 *추가 진입점* — 불안 상태 사용자에게 "이걸 봐야 하나?" 부담.
3. **메타포 충돌**: 디버깅·OS·항해 vs 콘텐츠 피드 = *책장·뉴스피드·라이브러리* 메타포 → 시각 일관성 깨짐.
4. **마이크로카피 회귀**: "친절히 설명" 어휘는 분석가 톤 위반 위험. CTA 카피("더 알아보기"·"이 글 읽기")가 *학습 강요*로 surface되면 자율성 미션 위반.
5. **접근성**: 긴 콘텐츠는 폰트·줄간격·키보드 네비 추가 검증 부담.

**product-designer 의견**: 별도 피드 형태는 **본질 위협 #2 직접 위반**. 단 *분석가 톤·refined minimalism·메타포 정합*을 만족하는 *기존 화면 내 학술 surface 강화*는 가능.

### 2.5 product-owner — PMF 측정 SOP·자발 언급 가드

📊 **측정 인프라 영향**:

- G2 (60일) 메트릭: 자발 언급 ≥30% / 30일 잔존 ≥15% / 결제 의향 ≥20%.
- 콘텐츠 피드는 *이 셋 어느 것도 강화하지 않음*. 오히려 자발 언급 측정에 노이즈:
  - 사용자: "콘텐츠 보고 알게 됐어요" — 우리 *유도된 어휘*인지 자발인지 모호 (`coding-rubric §0 #1` 위반 위험)
  - 인터뷰 코딩이 더 어려워짐
- analytics_events 추가 surface (콘텐츠 읽음·체류시간) — 측정 인프라 부담은 작지만 *측정 의미*가 약함.

**product-owner 의견**: G2 통과 *전* 도입은 **측정 신호 오염**. G2 통과 후 별도 hypothesis로 다루기.

### 2.6 senior-ux-researcher (신규) — 멘탈 모델·terminology fit

🧠 **사용자 멘탈 모델 분석**:

- 사용자가 BlueBird를 처음 만나는 onboarding 9 슬라이드는 **"이건 디버깅 도구다"** 정체성을 명확히 박음.
- *콘텐츠 피드*가 surface되면 사용자 멘탈 모델은 다음으로 분기:
  - (a) "디버깅 도구 + 보조 학습 자료" — 우리가 의도하는 분기.
  - (b) "콘텐츠 앱이 디버깅 도구도 같이 한다" — 카테고리 reframe.
  - (c) "둘 다 어중간하다" — 가치 제안 약화.
- **N=15+ 인터뷰 표본 없이는** (a)·(b)·(c) 분기 비율 예측 불가. 도입 전 검증 필요.

**Terminology fit**: "콘텐츠 피드"라는 어휘 자체가 SNS·미디어 카테고리. *분석가 톤*과 정합 어려움. 만약 도입한다면 라벨 어휘부터 재검토 필요(예: "분석 노트", "이론 인덱스", "참조 자료").

**senior-ux-researcher 의견**:
- 별도 피드 즉시 도입 → 멘탈 모델 위험 측정 안 됨. **거부**.
- 단 *콘텐츠 자체*는 사용자 학습에 가치 — **디버깅 사이클 *안에* 통합되는 형태**라면 멘탈 모델 강화 가능 (예: 분석 결과 페이지에서 *해당 왜곡 깊이 학습* 카드).

### 2.7 strategy-manager — 인터뷰 코딩 rubric 영향

📋 **자발 언급 가드 위반 위험**:

- 모집 공고·스크리닝 폼에 surface한 어휘는 인터뷰 자발 언급 카운트 X (`coding-rubric §0 #1`).
- 콘텐츠 피드를 도입하면 *피드에 surface한 모든 어휘*가 사용자 발화 노이즈가 됨:
  - 예: 피드에 "전망이론" 콘텐츠 surface → 사용자가 "전망이론이 흥미로웠어요" 인터뷰 답변 → *자발 언급으로 카운트 X* (유도됨)
- 즉 *학술 어휘 풀 자체가 측정 도구로서 무력화*될 위험.

**strategy-manager 의견**: G2 측정 후 도입. 베타 단계는 자발 언급 dictionary 보존이 우선.

### 2.8 risk-manager — 의료기기 표현·PIPA 가드

⚠️ **법적 리스크**:

- "친절히 설명" 콘텐츠 = 사용자에게 *조언*으로 해석될 위험.
- "이 왜곡이 있으면 이렇게 해보세요" 식 처방 어휘 → **표시광고법·의료기기법 위험**.
- 면책 카피가 콘텐츠 *각각*마다 surface돼야 함 (운영 부담).
- PIPA: 사용자 콘텐츠 소비 추적 시 *추가 동의* 필요 — privacy policy 갱신.

**risk-manager 의견**: 도입 시 콘텐츠 1건당 변호사 1차 검토 필요. G3 변호사 검토 *전*에 콘텐츠 양산은 위험.

### 2.9 senior-fullstack-engineer — 회귀 부담·기술 마이그레이션

🔧 **회귀 영향**:

- 신규 라우트 + 컴포넌트 → 회귀 보호 영역 확장.
- E2E 시나리오(`docs/qa/e2e-scenario-im1-prerequisite.md`)에 콘텐츠 페이지 추가.
- vitest·tsc·lint:copy 통과 확인 필요.
- **본 작업이 IM.1 모집 시작(5/6)을 지연**시킬 가능성.

**senior-fullstack-engineer 의견**: IM.1 모집 *후* 검토. 모집 전 신규 카테고리 빌드는 일정 risk 큼.

### 2.10 senior-qa-engineer — 독립 검증 라인

🔍 **회의록 정합성 점검**:

- CEO 발의문(§0)과 페르소나 검토 결과 비교: **9/10 페르소나가 *별도 피드 형태*에 거부 의사**, 1/10(senior-ux-researcher)이 *통합 형태*에 조건부 동의.

- 회의록 정합성 ✓: 결정에 대한 페르소나 발언이 각자 권한 경계 내.

**senior-qa-engineer 의견**: 별도 피드 형태 거부에 페르소나 합의 형성. 단 *콘텐츠 자체*의 가치를 살리는 *통합 형태*는 별도 검토 가치 있음.

## 3. 검토 합성 — Pros/Cons

|---|---|---|

### 3.2 Cons (별도 공간 형태로 도입 시)

| # | 항목 | 강도 |
|---|---|---|
| C1 | 본질 위협 #1·#2·#4·#6 다축 위반 가능성 | 高 |

**합성**: Pros는 *장기·미검증* 항목 위주, Cons는 *즉시·다축* 위반 가능성. 베타 단계 도입 비추.

## 4. 대안 4종 평가 (콘텐츠 가치 살리는 형태)

### 대안 A: 기존 자산 강화 (`/manual` + `/our-philosophy`)
- 현재 두 페이지에 학술 출처·이론 풀이 보강.
- **장점**: 신규 카테고리 0, 메뉴 추가 0, 본질 위협 위반 risk 0.
- **단점**: 콘텐츠 양·신선도 surface 어려움 (정적).
- **평가**: 즉시 적용 가능. PMF 단계 적합.

### 대안 B: 디버깅 사이클 *안*에 통합 — 분석 결과 페이지의 *왜곡 학습 카드*
- `/analyze/[id]`·`/visualize/[id]`에서 사용자에게 *지금 발견된 왜곡*에 대한 학습 카드 1~2개 surface.
- 학습이 *디버깅 동선 안*에 통합 → 카테고리 표류 0, 가치 제안 강화.
- **장점**: 사용자 멘탈 모델 강화 (디버깅 도구 정체성 ↑), Falsifiability 측정 가능 (학습 카드 노출 vs 비노출 잔존율 비교).
- **단점**: 구현 ~1 sprint, 본질 위협 #1 카피 점검 필수.
- **평가**: G2 통과 후 검토 — *즉시 도입 비추*. PMF 측정 가설 정의 후.

### 대안 C: 외부 채널 (Brunch @해솔 newsletter)
- 앱 내 카테고리 변경 X. CEO가 Brunch에 콘텐츠 발행.
- **장점**: 앱 카테고리 정합 0 risk. 외부 유입은 G3 후 자연 활성화.
- **단점**: 베타 단계 외부 마케팅은 명시적 비목표 → G3 후 트리거.
- **평가**: G3 후 활성. 단 *지금 글 자체를 쓰는 것*은 G2 측정과 무관해 가능.

### 대안 D: 무행동 + G2 후 재검토
- 현 상태 유지. IM.1 인터뷰에서 *학술 콘텐츠 자발 요청* 측정.
- **장점**: 사전 박지 않음. 데이터 기반 결정.
- **단점**: 사용자가 학습 자원 부족하다고 *말로 표현*하지 않으면 신호 누락.
- **평가**: 인터뷰 코딩 rubric에 신규 카테고리 추가로 보완 가능.
