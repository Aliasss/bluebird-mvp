## 1. CEO 개회 — 2가지 본질 질문 제기

오늘 P0-1·P0-2 구현으로 결과 깊이는 강화됐다. 그런데도 **"이 3단계만으로 충분한가"라는 질문이 남는다.** 인지 왜곡 자주 겪는 사람들의 공통 특성을 우리가 무시하고 있을 수 있다.

→ 두 가지 가능성 분기:
- (a) 3단계는 본질. 확장 없이 깊이만 강화 (P0-1·P0-2 방향)
- (b) 3단계는 본질이지만 너무 좁음. 인접 영역으로 확장 필요

## 2. 임원 라운드

### 2.1 CPO

3단계 chain 자체는 학술 근거 단단함 (Kahneman·Beck·Deci & Ryan). 문제는 **chain 진입 빈도**.

사용자가 불안 순간에 BlueBird를 떠올리는 빈도가 핵심 변수. 5/16 자기 인터뷰에서 C(트리거 인지 부재) +2로 2위였음. 푸시 인프라가 켜지면 이 부분은 해소되지만, **그래도 "왜 매번 이걸 써야 하나?"라는 동기 문제는 남는다.**

→ **CPO 견해: 3단계 본질은 유지하되, "인지 왜곡 + α" 영역의 α를 찾아야 한다.** α 후보는 인지 왜곡과 인과 관계 있는 공통 특성. 예: 반추-수면 / 결정 마비-회피 누적 / 신체화-스트레스 신호.

### 2.2 CTO

3단계 모두 구현 측면에서 합의가 단단. P0-1·P0-2로 단계 1·2 깊이는 보강됨. 단계 3(자율성) 깊이는 여전히 약함 — `autonomy_score`만 누적되고 사용자에게 surface 약함.

확장 관점: **인접 데이터 입력**(수면·운동·식사·SNS 사용)을 받으면 분석 변수가 풍부해져 단일 자동 사고 분석보다 깊이 있는 디버깅 가능. 다만 **입력 마찰 ↑** 위험 (5/16 인터뷰 A +1).

→ **CTO 견해: 확장은 가능하지만 입력 마찰 보존이 1차 제약. "사용자가 따로 입력 안 해도 시스템이 알아서 수집·연결" 방향이어야 한다.**

### 2.3 CSO

BlueBird 6축(톤·자기상·개입·시간·가격·메타포)에서 "인지 디버깅" 정체성은 디스턴싱·하루콩과 완전히 다른 자리를 점유. **확장으로 인해 이 자리를 흔들면 차별화 자체가 파괴된다.**

확장 가능 조건 — 모두 충족 시만 OK:
1. 본질 3단계가 약화되지 않을 것
2. "분석 도구·운영자 자기상·도구적 어휘" 톤이 유지될 것
3. 의료기기 함의(치료·진단·교정) 추가되지 않을 것
4. 인간 코치·외부 보상 시스템 추가되지 않을 것

→ **CSO 견해: 4 조건 충족하는 확장만 검토. 위반 시 거부.**

### 2.4 CMO

CMO Stage 0 유지 중. 확장 결정 전이라 마케팅 메시지에는 영향 없음. 다만 **확장하면 Stage 1 진입 시 카피·포지셔닝이 달라진다.**

현 BlueBird 한 줄 정의 "사고를 디버깅하는 도구"는 1차 타겟(분석가형)에게 강력. 확장 시 "사고 디버깅 + ○○○" 형태로 한 줄이 길어지면 분석가형에게 메시지 약해질 위험.

→ **CMO 견해: 확장하더라도 한 줄 정의는 "사고를 디버깅하는 도구" 유지. 확장은 *부가 가치*로만 전개 (가치 제안의 2층).**

## 3. 산하 라운드 — "인지 왜곡 자주 겪는 사람들의 공통 특성" 가설

### 3.1 senior-ux-researcher

학술 인용 자료에서 식별되는 **공통 특성 8가지** (Wells 2009, Aldao 2010, Struijs 2017 등):

| # | 특성 | 학술 근거 | BlueBird 정합 |
|---|---|---|---|
| 1 | **반추 (Rumination)** | Wells 2009 CAS | ✓ (감정 어휘 아님, 분석 어휘로 surface 가능) |
| 2 | **걱정 (Worry)** | Wells 2009 CAS | ✓ |
| 3 | **결정 마비 (Decision paralysis)** | Charpentier 2017 | ✓ (overthinking — 1차 타겟 정합) |
| 4 | **회피 누적** | Struijs 2017 9년 종단 | ✓ (이미 our-philosophy §08) |
| 5 | **반추-수면 영향** | Wells 2009, Carney 2015 | △ (신체 데이터 입력 필요) |
| 6 | **신체화 증상 (두통·소화·근긴장)** | 임상 보고 다수 | △ (의료 함의 위험) |
| 7 | **사회적 비교 민감 (SNS·동료)** | Festinger 1954, Vogel 2014 | ✓ (트리거 카테고리 확장) |
| 8 | **메타인지 함정 (지성화)** | Wells 2009, 2026-05-10 founder review | ✓ (이미 META_AVOIDANCE 가드) |

→ **senior-ux-researcher 견해: 5·6번은 신체 데이터·의료 함의 위험. 1·3·4·7번이 BlueBird 본질과 가장 정합하고 학술 근거도 단단함.**

### 3.2 strategy-manager

5/16 자기 인터뷰 F-1(rubric v0.1)에서 정의한 11개 트리거 카테고리(C1~C11) 중 현재 사용되는 건 8개(work·relationship·family·health·self·finance·study·other). 나머지 3개(C9~C11)는 보류 상태.

확장 후보를 **신규 트리거 카테고리**로 surface하면 인지 왜곡 본질을 깨지 않고 적용 폭만 넓힐 수 있음.

신규 카테고리 후보:
- **sleep_rumination** — "잠 안 옴" 트리거 + 자동 사고 (반추) 분석
- **decision_paralysis** — "결정 못 하겠음" 트리거 + 자동 사고 (이분법·파국화) 분석
- **social_comparison** — "다른 사람과 비교됨" 트리거 + 자동 사고 (개인화·임의적 추론) 분석

→ **strategy-manager 견해: 확장 = 신규 트리거 카테고리 추가. 본질 3단계는 그대로, 적용 영역만 확장.**

### 3.3 product-designer

UI 측면 — 확장이 화면 복잡도를 늘리면 안 됨. 현 분석 화면은 단일 자동 사고 입력 → 단일 결과. 신규 트리거 카테고리가 추가되어도 사용자 인지 부하는 변하지 않음 (카테고리는 메타데이터, UX는 동일).

다만 **수면·신체 데이터 입력 같은 별도 입력 화면 추가는 입력 마찰 ↑**. 거부 권장.

→ **designer 견해: 신규 카테고리 OK. 신규 입력 화면 ❌.**

### 3.4 senior-fullstack-engineer

기술 구현 측면 — 신규 트리거 카테고리 추가는 **마이그레이션 1건** 으로 가능. `logs.trigger_category` enum 확장 + `lib/ai/bluebird-protocol.ts` 분류 가이드 추가.

수면·신체 데이터 입력은 신규 테이블 + 분석 파이프라인 통합 필요 — 6~10시간 작업. IM.1과 충돌.

→ **senior-fullstack 견해: 신규 카테고리 = 작업 1~2h. 신규 데이터 입력 = 6h+. IM.1 후 격상이 합리적.**

### 3.5 senior-qa-engineer

회귀 위험 — 신규 카테고리 추가 시 기존 카테고리 분류 정확도가 영향받을 수 있음. AI 분류 회귀 테스트 fixture 확장 필요.

→ **senior-qa 견해: 신규 카테고리 1개당 fixture 3~5건 추가 필요. 작업 자체보다 검증 부담 더 큼.**

### 3.6 data-analyst

확장 효과를 측정하려면 IM.1 베타 데이터 누적 후 비교 가능. 지금 결정하기엔 **데이터가 0건**. IM.1 종료 후 다음 패턴 분석:

- 사용자 자발 입력 트리거에서 "수면" "결정" "비교" 어휘 등장 빈도
- 자발 등장 비율이 ≥15%면 → 신규 카테고리 가치 있음
- ≤5%면 → 본질 3단계 강화만으로 충분

→ **data-analyst 견해: 확장은 IM.1 D14 데이터 후 결정. 지금은 측정 불가.**

### 3.7 risk-manager

수면·신체화·결정 마비 같은 영역은 **임상 진단명과 인접 (예: 불면증·신체증상장애·강박)**. 카피·기능에서 의료 함의 검출되면 즉시 차단 필요.

신규 트리거 카테고리는 OK (단순 메타데이터). 그러나 카테고리 이름에 "장애·증상·진단" 어휘 금지.

→ **risk-manager 견해: 신규 카테고리명은 "어려움·상황·맥락" 어휘로만. 임상 어휘 ❌.**

### 3.8 content-marketer

신규 트리거 카테고리는 카피에 영향 0 (사용자가 직접 트리거 입력하므로). 그러나 새 카테고리가 surface되면 인사이트 페이지·매뉴얼에 새 어휘가 추가됨. 분석가 톤 유지 필수.

→ **content-marketer 견해: 카테고리 추가는 lexicon 영향 작음. 추가 시 lexicon v2 갱신.**

### 3.9 performance-marketer

IM.1 모집 카피·인터뷰 가이드(F-10)에 신규 트리거 카테고리 후보 어휘를 미리 묻는 질문 1~2개 추가 가능. 데이터 수집을 모집 단계부터 시작.

→ **performance-marketer 견해: F-10 인터뷰 가이드에 "최근 가장 자주 떠올린 어려움 1가지"를 자유 입력으로 묻는 1문항 추가 권장.**

## 4. 핵심 합의 — Cross-functional 결론

### 4.1 본질 3단계는 변경 없음

| 단계 | 변경 여부 |
|---|---|
| 1. 인지 왜곡 탐지 | **유지** (P0-1·P0-2로 깊이 강화 완료) |
| 2. 시스템2 활성화 질문 | **유지** (현 소크라테스 흐름 유지) |
| 3. SDT 기반 자율성 | **유지** (단계 3 깊이 강화는 별도 후속 — P0-3 보류 상태) |

### 4.2 확장의 정의 — "신규 트리거 카테고리"

확장 = **인지 왜곡 디버깅의 적용 영역을 넓힘**. 본질 3단계는 동일하게 작동.

후보 카테고리 (IM.1 데이터로 검증):

| 후보 | 자발 언급 임계 | 학술 근거 |
|---|---|---|
| sleep_rumination (잠 안 옴 + 반추) | 자발 ≥15% | Wells 2009 CAS |
| decision_paralysis (결정 못 하겠음 + overthinking) | 자발 ≥15% | Charpentier 2017 |
| social_comparison (SNS·동료 비교 + 개인화) | 자발 ≥15% | Vogel 2014 |
| avoidance_accumulation (회피 누적) | 자발 ≥15% | Struijs 2017 |

### 4.3 거부된 확장 방향

| 방향 | 거부 사유 |
|---|---|
| 수면·운동·식사 신체 데이터 입력 | 입력 마찰 ↑ (CTO·designer), 의료 함의 위험 (risk-manager) |
| 인간 코치 매칭 | 본질 위협 #2 (인간 동반 회귀), CSO 4 조건 위반 |
| 외부 보상·게임화 | SDT 위반 (외부 동기가 내재 동기 잠식, Deci & Ryan 1985) |
| 감정 일기·기분 트래킹 | 본질 위협 #1 (감성 챗봇 회귀), 카테고리 정체성 파괴 |

### 4.4 IM.1 기간 측정 설계

5/15 weekly all-hands F-10(인터뷰 가이드 v1) 작업에 다음 1문항 추가 권장 (F-10 작업자 = senior-ux-researcher):

> "최근 2주 동안 가장 자주 떠올린 어려운 상황 1가지를 짧게 적어주세요. (예: '발표 망쳤다'·'잠 안 옴'·'결정 못 하겠음'·'친구와 비교됨' 등)"

응답 코딩으로 위 4 후보 카테고리 자발 등장률 측정 → IM.1 D14 결정 입력값.
