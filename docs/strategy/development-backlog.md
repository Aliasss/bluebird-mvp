## 결제 가설 (모든 Tier가 이걸 강화하기 위해 존재)

CBT 앱 사용자가 돈을 내는 이유는 셋 중 하나:

- **A. "내가 변하고 있다는 증거"** — Δpain·자율성·패턴 인식이 시각화될 때
- **B. "이걸 멈추면 후퇴한다"** — 데이터 락인 + 습관
- **C. "한 번의 결정적 통찰"** — 무료에서 받은 강한 경험 → 자발적 결제

**블루버드의 핵심은 A.** B·C는 보조. 다른 정신건강 앱과의 진짜 차별점이 "측정 가능한 인지 변화"이기 때문.

- **AI 챗봇화** — Wysa·Replika 카테고리. 우리는 측정 도구, 그건 우리가 못 이기는 싸움.
- **무료 사용자 알림 폭격** — 정신건강 앱에서 푸시 남발 = 즉시 삭제. 1.4도 *맥락이 명확한 알림만*.
- **PMF 검증 전 결제 인프라** — 2.3은 2.2 통과 후에만.
- **하드코딩된 소크라테스 질문 풀** — Gemini가 동적 생성하는 게 개인화 측면에서 우월. 도메인 가이드는 프롬프트로 흡수.

## 🚧 리스크 (결정 시 항상 참조)

### 의료기기/의료법 리스크
- "치료" 어휘 절대 X. "자가 인지 코칭 도구" 톤 유지.
- 면책 페이지 강화 필요 (현재 부족).
- 전문의 자문 1명(3.3)이 있으면 회색지대 방어 ↑.

### 한국 정신건강 stigma
- 구전이 어려움 → 주간 리포트(2.1)로 *결과물*을 공유하게 만드는 우회 전략.
- 익명성 보장 강조.

### 데이터 신뢰
- 결제 받기 시작하면 "내 정서 데이터를 누가 보나" 우려 즉시 발생.
- 프라이버시 정책 + 암호화 명시 + 데이터 다운로드/삭제권 필요.

### Gemini API 의존도
- 분석 품질이 Gemini Flash 성능에 직접 묶여 있음.
- 모델 deprecation / 가격 변경 / 정책 변경 리스크.
- 장기적으로 멀티 프로바이더 fallback (Claude Haiku 등) 검토.
