## 0. 정의 — PMF란?

본 프로젝트에서 PMF는 다음 셋이 *동시에* 만족될 때 인정한다.

1. **자발 가치 언급** — 사용자 인터뷰에서 *우리가 유도하지 않은* 핵심 가치(특히 "패턴이 보인다", "변화를 느낀다")가 ≥30% 자발적으로 나옴
2. **잔존 임계 통과** — 30일 잔존율 ≥15% (한국 정신건강 앱 평균 5~10% 대비 1.5~3배)
3. **결제 의향 시그널** — 인터뷰에서 결제 의향 사용자 비율 ≥20%, 가격 분포가 인터뷰 결과로 *자연스럽게 모임*

이 세 조건 미달 = PMF 미확인 = 결제 인프라(2.3) 진입 X. 가설 재검토.

## 1. 핵심 가설 (검증 대상)

### 가설 A — "측정 가능한 인지 변화"가 결제 동인이다
사용자가 본인의 *변화*를 Δpain·자율성 그래프·패턴 리포트로 *눈으로 볼 때* 결제 의향이 발생한다.

### 가설 B — "데이터 락인"이 30일+ 잔존을 만든다
패턴 리포트는 데이터 누적 = 가치 증가 모델. 사용자가 *내 사용설명서*를 만들고 있다는 감각이 후퇴 회피 동기를 만든다.

### 가설 C — "한국어 분석 깊이"가 카테고리 창출 자산이다
한국어 우회 어미 처리 같은 fine-tuning이 글로벌 앱과의 *지속 가능한 차별점*이다.

### 가설 D — "분석직 세그먼트에서의 차별화"가 디스턴싱 흡수를 막는다
(2026-04-28 추가, 경쟁 전략 v1 §5 가설 재정의 통합)

분석직·인지노동자(BA, PM, 엔지니어, 컨설턴트, 연구자) 세그먼트에서 BlueBird가
디스턴싱·기타 멘탈케어 앱 대비 *선호되는* 신호가 있어야, 후발주자로서 흡수되지
않는다. 검증 질문: BlueBird만의 결합(분석적 톤 + 한국어 NLP + 자율성 회복 미션)이
이 세그먼트에서 직관적으로 받아들여지는가?

각 가설은 다음 마일스톤에서 검증된다.

## 4. 60일 결정 게이트

이 시점에 다음을 평가하고 *결정*을 내린다.

### 통과 조건 (전부 충족 시 Tier 2 진입)
- [ ] 30일 잔존율 ≥15%
- [ ] 인터뷰 자발 가치 언급률 ≥30%
- [ ] 결제 의향 사용자 비율 ≥20%
- [ ] 임상 자문 후보군 ≥1명 발굴 (3.3 시작 가능)

### 부분 통과 (보강 작업)
- 잔존율은 OK이나 결제 의향 약함 → 가치 *명료성* 강화 (UX/메시지 작업)
- 자발 언급 OK이나 잔존 약함 → 락인 메커니즘 강화 (PWA Push, 마일스톤)
- 둘 다 약함 → 미통과로 처리

### 미통과 (가설 재검토)
- 사용자가 *말하는 가치*와 *우리가 만든 가치*가 다른가?
- 어떤 카테고리(트리거 도메인)에서만 작동하는가? — 그쪽으로 좁힐 가능성
- 사용 패턴이 우리 가설과 다른가? — 피벗 시그널

피벗 옵션 (사전 정의 — 결정을 미루지 않기 위해):
1. **B2B 피벗** — 기업 EAP·복지 패키지 (개인 결제 대신 법인 결제)
2. **임상 협력 피벗** — 정신과 의원 보조 도구로 포지션
3. **수직 좁힘** — 특정 카테고리(예: 직장인 발표 불안)에 집중

피벗 결정도 60일 게이트에서 함께 검토.

## 7. 가장 큰 미지수 5개 (해소 전 결제 진입 X)

1. **PMF 시그널** — 우리 가설이 사용자 가설과 일치하는가, 아니면 우리가 *만든* 가설인가?
2. **잔존율 현실** — 한국 정신건강 앱 평균(<10%) 대비 우리는 어디인가?
3. **결제 의향 임계** — 인터뷰가 *얼마*를 자발적으로 답하는가? (사전 박지 않음)
4. **구전 메커니즘** — Stigma를 우회하는가? — 주간 리포트(2.1)가 답일 수도
5. **임상 자문 확보 가능성** — 전문가가 *AI 도구*에 자문 붙을 의향이 있는가?

## 9. 결정 권한 (의사결정 흐름)

각 게이트 통과/미통과 결정은 알빈(파운더) 단독.
판단 자료는 다음 셋:
1. 정량 데이터 (메트릭)
2. 정성 데이터 (인터뷰 코딩)
3. 본 문서의 *사전 정의 임계*

CPO/CTO(Claude)는 데이터 기반 권고만 제공. 게이트 통과 임계를 사후에 *조정*해서 통과시키는 패턴(motivated reasoning)은 *명시적으로 회피*. 임계가 미달이면 미달.

2. **차별화 정합성** (디스턴싱과 구별되는가) — 신규
3. 미션 정합성 (자율성 회복)
4. 사용자 인지 부하

| 모듈 D(신체화 매핑) 차별화 축 격상 | §8 명시적 비목표에 deferred | 인터뷰 자발 언급률 ≥10% 시 → MVP 진입 검토. 미만이면 옵션 C 유지. |
| 가격 가설 사전 박음 (월 1.9~3.9만원) | §0 PMF 정의의 "가격은 인터뷰로 자연 수렴" 원칙 | 인터뷰 결제 의향 분포가 1.9~3.9만 구간에 자연 수렴하면 가설 채택. 사전 박지 않음. |
| 법적 표현 정렬 ("탐지" → "기록·관찰") | 코드 전반 라벨 변경 비용 + 디지털의료제품법 적용 가능성 변호사 검토 미진행 | §8 명시적 비목표(외부 마케팅 활성 G3 이전엔 검토 안 함). 단, 법적 리스크 신호 감지 시 즉시 트리거. |

### 11.4 의문 제기 — 강한 단정에 대한 CPO 코멘트

경쟁 전략 v1이 단정한 다음 두 명제는 *데이터 없이는 받아들이지 않는다*:

1. **"분석가 톤만이 차별화 가능 영역"** — 현재 BlueBird의 *분석적 항해사*
   메타포(나침반 = 데이터 기반 의사결정, 항해 = 불확실성 하의 합리적 항로)는
   디스턴싱의 *거리두기·치유* 톤과 명확히 다르다. 하이브리드 포지션 가능성을
   배제할 만한 데이터가 아직 없다.

2. **"신체화 매핑이 단독 점유 가능한 메커니즘 자산"** — 사용자가 그것을 *가치로
   인식*하는지, 신체 데이터 입력 부담을 감수하는지는 검증되지 않은 가설. 인터뷰
   자발 언급으로만 시그널 잡을 수 있다.

### 11.5 트리거 — 본 부록의 활성화 시점

본 부록의 보류 항목·의문 제기는 60일 게이트(§4) 시점에 *반드시* 재방문한다.
재방문 입력:
- §3 M30.1 인터뷰 결과 (차별화 코딩 포함)
- §6 메트릭 (직군 분포, 경쟁앱 경험자 평가, 톤 인식)
- 누적 분석 품질 메트릭 (`analytics_events` 테이블)

재방문 결과는 본 부록 §11.3 표를 직접 갱신한다.
