## 0. 작동 메커니즘 — 3단계 목적 (창업자 정의, 2026-05-16 아카이빙)

BlueBird가 사용자에게 하는 일을 3단계로 분해. 이 정의는 **모든 기능·카피·디자인 결정의 1차 정합성 기준**이다. 새 기능 제안 시 다음 질문에 답할 수 있어야 한다: "이 기능이 1·2·3단계 중 어느 단계를 강화하는가?"

### 단계 1 — 인지 왜곡 탐지

**무엇:** 뇌의 정보 처리 과정에서의 편향(인지 왜곡)을 자동으로 식별한다.

**근거:** Tversky & Kahneman (1992) Prospect Theory · Beck (1976) Cognitive Therapy of Depression · Wells (2009) Metacognitive Therapy.

**현 구현:** `lib/ai/bluebird-protocol.ts` + Gemini multi-distortion 출력 (5가지: 파국화·이분법·감정적 추론·개인화·임의적 추론).

### 단계 2 — 시스템2 활성화 질문

**무엇:** 탐지된 왜곡을 사용자가 *스스로* 인지하도록 하고, 인지 후 *분석 모드(시스템2)*가 자동 가동되도록 소크라테스식 질문을 던진다.

**근거:** Kahneman (2011) *Thinking, Fast and Slow* — 시스템1(직관·자동·왜곡 쉬움) vs 시스템2(논리·분석·왜곡 교정). BlueBird는 시스템1 오류를 surface하고 시스템2를 강제 기동하는 장치.

**현 구현:** `app/api/generate-questions/route.ts` + `app/analyze/[id]/page.tsx` 소크라테스 3문항 흐름.

### 단계 3 — 자율성 부여 (자기결정이론 기반)

**무엇:** 사용자가 *자발적으로* 행동 계획을 수립·실행·완료함으로써 자율성을 회복한다. 정답 제시 ❌, 행동 강요 ❌, 외부 보상 ❌.

**근거:** Deci & Ryan (2000) Self-Determination Theory (SDT) — autonomy 차원이 지속 동기의 핵심. 외부 통제·보상이 내재 동기를 잠식.

**현 구현:** `lib/intervention/autonomy-score.ts` v2 + `app/action/[id]/page.tsx` (최종 행동 작성 + 완료 여부 + 완료 노트).

### 본질 정의 (3단계 통합)

> **BlueBird = 인지 왜곡 자동 탐지 → 시스템2 강제 기동 → SDT 기반 자율성 회복.**
>
> 이 3단계 chain의 어느 한 단계라도 빠지거나 약화되면 BlueBird가 아니다. 새 기능은 이 3단계를 **깊게(강화)** 또는 **넓게(인접 적용)** 만들어야 한다. 본질을 우회·대체하는 기능은 거부 (예: 위로·코치 매칭·외부 보상 시스템).

### 확장 가능성 (2026-05-16 deep dive surface)

3단계가 본질이지만, **인지 왜곡을 자주 겪는 사람들의 공통 특성**(수면·반추·결정 마비·신체화·회피 누적 등) 중 본 3단계와 정합하는 영역은 *확장* 후보. 후보 검토는 `docs/meetings/2026-05-16-purpose-and-expansion-deep-dive.md` 참조.

## 1. 카테고리 포지셔닝

| 우리는 *아닌* 것 | 점유자 |
|---|---|
| 스트레스 관리 앱 | 디스턴싱·하루콩·인사이드 |
| 명상·힐링 | 마보 |
| 인간 코치 매칭 | 디스턴싱·트로스트·마인드카페 |
| 의료기기·DTx | 솜즈·ANZELAX·Cogthera (식약처 트랙) |
| 정서 챗봇 | Wysa·Woebot (글로벌, 한국어 미진입) |

**우리가 *맞는* 것:**
- 인지 왜곡 디버깅 도구 (Cognitive Distortion Debugging Engine)
- 자가 인지 코칭 (도구 + 측정 + 패턴 누적)
- *분석가형 자기이해 선호자*의 1번 도구
- 일반 웰니스 (의료기기 X)

## 2. 타겟 세그먼트

**1순위 (PMF 검증 대상): 1차 타겟 3축 동시 충족자**

직무 무관. 다음 3축이 동시에 충족되는 사용자:
- **심리·동기**: 감정 언어가 부담스럽고 자기를 *시스템처럼* 이해하고 싶음
- **인지 스타일**: 분석적 자기이해 선호 — "위로받기"보다 "구조 보기"
- **맥락**: 한국 인지노동자 (직장·체면·자기낙인 맥락)

각 축은 *독립 측정 가능* — 인터뷰 자발 언급 코딩으로 검증. 직무 enumerate(BA·PM·엔지니어·컨설턴트·연구자 등)는 *예시일 뿐 한정 X*. 측정은 직무 분포가 아니라 3축 자발 언급으로.

**2순위 (확장 후보, G3 이후):** 자기성찰 학습자, 자기계발 지향 30대 직장인

## 3. 가치 제안

| 카테고리 | 내용 | 검증 메커니즘 |
|---|---|---|
| **핵심 가치 (Δ 측정)** | 사고 디버깅 후 *측정 가능한 통증 감소* | `Δpain` 메트릭 (PMF plan §6) |
| **누적 가치 (락인)** | 내 사용설명서 (패턴 리포트) — 사용할수록 정교해지는 자기 지도 | 30일+ 사용자에게 통증·왜곡 상관관계 surface |
| **차별 가치 (카테고리 우위)** | 한국어 fine-tuning (우회 어미·체면 어휘) + 분석가 톤 | 인터뷰 자발 가치 언급 코딩 |

> **누적 가치 락인 메커니즘**: [`bluebird_retention_mechanisms_v1.md`](./bluebird_retention_mechanisms_v1.md) — *내 사용설명서*(매뉴얼)와 *익명 통계 맥락화* 메커니즘으로 구체화. 베타 단계는 데이터 스키마만 / G2 통과 후 매뉴얼 v1.0 / G3 통과 후 통계 노출.

## 4. 차별화 3축 (vs 디스턴싱)

| 축 | 디스턴싱 | BlueBird |
|---|---|---|
| **톤** | 정서적·치유적 ("따뜻한 봄날", "외롭지 않게") | 분석적·구조적 ("분석가 정밀함", "구조 가시화") |
| **사용자 자기상** | "회복이 필요한 사람" | "최적화하려는 운영자" |
| **개입 방식** | 인간 코치 매칭 + 활동지 | 자동화 + 반증 질문 + 행동→재평가 회로 |
| **시간 모델** | 3개월 권장 (체화) | 90일 검증 (가설 falsifiability) |
| **가격 가설** | 월 8~10만원 (코치 인건비) | 월 1.9~3.9만원 (자동화) — 인터뷰 후 결정 |
| **메타포** | 거리두기·치유·동반 | 디버깅·OS 최적화·항해 |

## 5. 이론적 백본

1. **CBT** (Beck) — 인지 왜곡 5종 분류 (`bluebird-protocol.ts`에 코드화)
2. **Dual Process** (Kahneman, *Thinking, Fast and Slow*) — System 1/2 활성화 (10ms vs 3s)
3. **Prospect Theory** (Kahneman & Tversky 1979) — 손실 회피 λ ≈ 2.25
4. **Metacognition / CAS** (Wells) — 반추·걱정 신호 측정
5. **Cognitive Biases** (Heuer, *Psychology of Intelligence Analysis*) — 분석가 톤의 학술 근거
6. **실존주의** (Camus, *Myth of Sisyphus*) — 미션 "자율성 회복"의 철학적 배경

(catalog: [`docs/research/README.md`](../research/README.md))

## 6. 미션·비전

**미션 (단기 — 90일):**
한국 인지노동자가 *자기 사고를 더 정확하게 보는* 1번 도구가 된다. 디스턴싱이 점유하지 않은 분석가 세그먼트에서 PMF 시그널 확보.

**비전 (중기 — 12~24개월):**
"한국어 인지 디버깅"이라는 *카테고리 자체를 창출*. 글로벌 AI 챗봇(Wysa·Woebot) 한국어 진입 시도에도 한국 직장 문화·체면·자기낙인 맥락은 단기간 따라잡기 어렵게 락인.

**비전 (장기):**
분석적 항해사 정체성을 가진 사용자가 *자기 자신을 디버깅*하고 *자율성을 회복*하는 도구. 치료가 아닌 *자기 운영*.

## 7. PMF 정의 (3가지 동시 충족 시 인정)

| 임계 | 측정 |
|---|---|
| 자발 가치 언급 ≥30% | 인터뷰에서 *우리가 유도하지 않은* 핵심 가치 자발 언급 |
| 30일 잔존율 ≥15% | 한국 정신건강 앱 평균(<10%) 대비 1.5~3배 |
| 결제 의향 ≥20% | 인터뷰 결제 의향 사용자 비율, 가격 분포 자연 수렴 |

**현재 위치 (2026-04-29):** 베타 사용자 0명, IM.1 모집 시작 직전. 다음 게이트 60일 → 90일 → G3 외부 진입.

## 8. 명시적 비목표

베타 단계에서 *하지 않는* 것 (PMF plan §8 + §11.3):

| 항목 | 트리거 |
|---|---|
| 결제 인프라 | G2 통과 후 |
| 외부 마케팅 | G3 통과 후 |
| 모바일 네이티브 | PWA 검증 후 |
| 다국어 | 한국 시장 검증 후 |
| 모듈 D 신체화 매핑 격상 | 인터뷰 자발 언급 ≥10% |
| 분석가 톤 전면 마이그레이션 | 인터뷰 톤 인식 분포 |
| 라벨 변경 ("통증" vs "스트레스") | 인터뷰 자발 언급 ≥60% 우세 |
| 법적 표현 정렬 ("탐지" → "기록·관찰") | G3 변호사 검토 후 |
| 다크모드 | 인터뷰 자발 언급 ≥10% |
| 챗봇 카테고리 | 영구 비목표 (못 이기는 싸움) |

## 9. 본질 위협 신호 (즉시 멈춤 트리거)

다음 발생 시 *작업을 멈추고* [`bluebird_competitive_strategy_v1.md`](./bluebird_competitive_strategy_v1.md) §6.4 + [`pmf-validation-plan.md`](./pmf-validation-plan.md) §11.6으로 복귀:

1. 카피가 정서적·치유적 톤으로 자연스럽게 흘러감 ("함께", "위로", "마음을 안아")
2. 디자인이 파스텔·자연 사진·감성 일러스트로 수렴
3. "스트레스 관리"가 *입구*로 등장 (메뉴·카피·랜딩)
4. 정기 자기 라벨링 체크인 (모닝/이브닝 알림 등)이 추가됨
5. 가격 책정이 디스턴싱과 비슷한 수준이 자연스럽게 느껴짐
6. 기능 기획 동기가 "사용자가 외롭지 않게" 류로 흐름

## 10. 의사결정 우선순위

미시 결정에서 충돌이 생기면 다음 순서로 적용:

1. **검증 가능성** (Falsifiability) — 데이터 신뢰도를 해치는가?
2. **차별화 정합성** — 디스턴싱과 구별되는 BlueBird 정체성 강화/약화?
3. **미션 정합성** — "자율성 회복"과 충돌하는가?
4. **사용자 인지 부하** — 불안 상태 사용자의 인지 부하 증가?
5. **구현 비용** — 위 넷이 동등하면, 더 적은 비용 선택.
